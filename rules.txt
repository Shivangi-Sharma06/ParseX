PRE-REQUISITES:- Here are the key prerequisites for developing a Full Stack MERN (MongoDB, Express, React, Node.js) application with JWT Authentication: 1. Node.js and npm Node.js is a JavaScript runtime environment that allows you to run JavaScript on the server side. npm (Node Package Manager) helps install and manage dependencies. ● Install Node.js and npm on your machine 
● Verify installation: Shell node -v npm -v 2. MongoDB (Database) MongoDB is a NoSQL database used to store application data in JSON-like format.
 ● Install MongoDB locally OR use MongoDB Atlas (cloud database) 
 ● Install MongoDB Compass (optional GUI tool) 3. React.js (Frontend) React is used to build dynamic and reusable UI components. Create a React App using Vite: Shell npm create vite@latest ●
  Enter project name ● Select React Shell cd project-name npm install npm run dev 4. Express.js & Node.js (Backend) Express is a lightweight backend framework used to build APIs. Initialize backend: Shell mkdir backend cd backend npm init -y 5. Basic Knowledge Required ● HTML – Structure of web pages ● CSS / Tailwind CSS – Styling and responsiveness
   ● JavaScript (ES6+) – Core programming concepts ● React Basics – Components, Props, State, Hooks ● REST APIs – GET, POST, PUT, DELETE ● Node.js – Server-side JavaScript runtime environment
    ● Express.js – Backend framework for building REST APIs ● MongoDB – NoSQL database for storing application data ● Mongoose – ODM for schema design and database interaction ● REST API Development – Creating APIs (GET, POST, PUT, DELETE)
     ● Middleware – Request/response handling (auth, logging, error handling) ● Authentication & Authorization – JWT-based login and protected routes ● Password Security – Hashing using bcrypt ● Environment Variables – Managing secrets using dotenv
      ● Error Handling – Proper status codes and API responses 6. Authentication (JWT) JWT (JSON Web Token) is used for secure authentication. ● Understand: ○ Token generation ○ Token verification ○ Middleware protection ○ Login / Signup flow 7. Version Control Use Git for tracking and collaboration. Shell git init git add . git commit -m "Initial commit"
       8. Development Environment Recommended tools: ● Visual Studio Code ● Postman (API testing) ● MongoDB Compass PROJECT STRUCTURE Frontend (React) None frontend/ │── src/ │   │   │   │   │   │   ├── components/ ├── pages/ ├── services/ (API calls) ├── routes/ ├── App.jsx ├── main.jsx │── package.json Backend (Node + Express) None backend/ │── controllers/ │── models/ │── routes/ │── middleware/ │── config/ │── server.js │── package.json MINIMUM REQUIREMENTS ● Frontend Styling ○ Use Tailwind CSS (or any UI library like Chakra UI / MUI / ShadCN) for a clean, responsive user interface. ● API Communication ○ Use Axios or Fetch API to handle communication between frontend (React) and backend (Node.js + Express). ● Routing ○ Implement React Router DOM for navigation across multiple pages (Login, Register, Dashboard, etc.). ● Backend Development ○ Build a backend using Node.js + Express.js. ○ Follow proper MVC architecture: ■ Models ■ Controllers ■ Routes ● Database Integration ○ Use MongoDB for storing application data. ○ Use Mongoose for schema design and database operations. 
       ● Authentication & Security ○ Implement JWT Authentication: ■ User Registration & Login ■ Token generation and verification ○ Use bcrypt for password hashing ○ Protect private routes using middleware ● Full CRUD Functionality Application must support: ○ Create – Add new data ○ Read – Fetch/display data ○ Update – Modify existing data ○ Delete – Remove data 
       ● State Management (Optional but Recommended) ○ Use Context API / Redux / Zustand for managing global state (user session, auth state) ● Error Handling ○ Proper API responses with status codes ○ Frontend error handling (toast notifications / alerts) ● Environment Configuration ○ Use .env file to store: ■ Database URI ■ JWT Secret ■ Port number RECOMMENDED NPM PACKAGES Frontend (UI & Utilities)

User registers → Data stored in MongoDB (hashed password) 
        2. User logs in → JWT token generated 3. Token sent to frontend and stored (localStorage/cookies) 4. Protected routes require token verification 5. User performs CRUD operations via API USER FLOW ● Register → Login ● Redirect to Dashboard ● Perform CRUD operations ● Logout (token removed) OPTIONAL ENHANCEMENTS ● Role-based authentication (Admin/User) ● File upload (Cloudinary / Multer) ● Pagination & search ● Dark mode UI ● Deployment (Vercel + Render / AWS) Note:- must follow the minimum requirements to successfully accomplish the project.
